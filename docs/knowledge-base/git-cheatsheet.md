# Шпаргалка по командам Git (табличный формат)

## ⚙️ Настройка

| Команда | Описание |
|---------|----------|
| `git config --global user.name "Имя"` | Установить имя пользователя |
| `git config --global user.email "email"` | Установить email |
| `git config --list` | Показать текущие настройки |

## 🆕 Создание и клонирование

| Команда | Описание |
|---------|----------|
| `git init` | Инициализировать новый репозиторий |
| `git clone <url>` | Склонировать удалённый репозиторий |
| `git clone -b <ветка> <url>` | Склонировать определённую ветку |

## 📝 Основная работа

| Команда | Описание |
|---------|----------|
| `git status` | Показать состояние файлов |
| `git add <файл>` | Добавить файл в staging area |
| `git add .` | Добавить все изменения в текущей папке |
| `git add -A` | Добавить абсолютно все изменения |
| `git commit -m "сообщение"` | Создать коммит |
| `git commit -a -m "сообщение"` | Пропустить `git add` (для отслеживаемых) |
| `git restore <файл>` | Отменить изменения в рабочей папке |
| `git restore --staged <файл>` | Убрать файл из staging |
| `git rm <файл>` | Удалить файл и добавить удаление в staging |
| `git mv <старое> <новое>` | Переименовать файл |

## 📜 История и просмотр

| Команда | Описание |
|---------|----------|
| `git log` | Показать историю коммитов |
| `git log --oneline` | Краткий вид (хеш + сообщение) |
| `git log --graph --oneline --decorate` | Граф веток |
| `git diff` | Разница между рабочей папкой и staging |
| `git diff --staged` | Разница между staging и последним коммитом |
| `git diff <коммит1> <коммит2>` | Разница между двумя коммитами |
| `git show <хеш>` | Показать изменения конкретного коммита |

## 🌿 Ветвление и слияние

| Команда | Описание |
|---------|----------|
| `git branch` | Список локальных веток |
| `git branch -r` | Список удалённых веток |
| `git branch -a` | Список всех веток |
| `git branch <имя>` | Создать новую ветку |
| `git checkout <ветка>` | Переключиться на ветку |
| `git switch <ветка>` | Переключиться (новый синтаксис) |
| `git checkout -b <ветка>` | Создать и переключиться |
| `git merge <ветка>` | Влить ветку в текущую |
| `git branch -d <ветка>` | Удалить локальную ветку (слитую) |
| `git branch -D <ветка>` | Принудительно удалить ветку |

## 🌍 Удалённые репозитории

| Команда | Описание |
|---------|----------|
| `git remote -v` | Показать удалённые репозитории |
| `git remote add origin <url>` | Добавить удалённый репозиторий |
| `git push -u origin main` | Отправить ветку и установить upstream |
| `git push` | Отправить изменения |
| `git push origin <ветка>` | Отправить конкретную ветку |
| `git pull` | Забрать + смержить (fetch + merge) |
| `git fetch` | Забрать изменения (без слияния) |

## 🔄 Отмена изменений

| Команда | Описание |
|---------|----------|
| `git reset HEAD~1` | Отменить коммит, сохранив изменения |
| `git reset --hard HEAD~1` | Полностью удалить последний коммит (⚠️ опасно) |
| `git reset --hard <хеш>` | Откатиться к коммиту (всё после - пропадёт) |
| `git revert <хеш>` | Создать коммит, отменяющий указанный (✅ безопасно) |
| `git commit --amend -m "новое"` | Изменить сообщение последнего коммита |
| `git commit --amend --no-edit` | Добавить файлы в последний коммит (перед этим `git add`) |

## 💾 Временное сохранение (Stash)

| Команда | Описание |
|---------|----------|
| `git stash` | Спрятать незакоммиченные изменения |
| `git stash save "описание"` | Спрятать с описанием |
| `git stash list` | Показать список спрятанного |
| `git stash apply` | Применить последний stash (оставить в списке) |
| `git stash pop` | Применить и удалить из списка |
| `git stash drop` | Удалить последний stash |

## 🔍 Диагностика

| Команда | Описание |
|---------|----------|
| `git status` | Главная команда — используйте постоянно! |
| `git log --oneline --graph --all` | История всего репозитория графом |
| `git blame <файл>` | Кто и когда менял каждую строку |
| `git grep "текст"` | Поиск текста по репозиторию |
| `git remote show origin` | Информация об удалённом репозитории |

## 🧹 Уборка

| Команда | Описание |
|---------|----------|
| `git clean -n` | Показать, какие неотслеживаемые файлы будут удалены |
| `git clean -f` | Удалить неотслеживаемые файлы (⚠️ осторожно) |
| `git gc` | Очистка и оптимизация репозитория |

## 🧠 Типовой рабочий процесс (по шагам)

| Шаг | Команда | Что делает |
|-----|---------|-------------|
| 1 | `git pull` | Синхронизация с удалённым репозиторием |
| 2 | `git checkout -b feature` | Создать новую ветку для задачи |
| 3 | `git add .` | Добавить изменения |
| 4 | `git commit -m "message"` | Создать коммит |
| 5 | `git push -u origin feature` | Отправить ветку на сервер |
| 6 | `git checkout main` | Вернуться в основную ветку |
| 7 | `git pull origin main` | Обновить main |
| 8 | `git merge feature` | Влить изменения |
| 9 | `git push origin main` | Отправить обновлённый main |
| 10 | `git branch -d feature` | Удалить локальную ветку |

## 🎯 Быстрые псевдонимы (альясы)

| Команда для настройки | Результат |
|-----------------------|-----------|
| `git config --global alias.co checkout` | `git co` |
| `git config --global alias.br branch` | `git br` |
| `git config --global alias.ci commit` | `git ci` |
| `git config --global alias.st status` | `git st` |
| `git config --global alias.unstage 'restore --staged'` | `git unstage` |
| `git config --global alias.last 'log -1 HEAD'` | `git last` |
